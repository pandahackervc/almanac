1. Axioms

Representation - (How information is represented)

All information is encoded as finite, discrete symbols (e.g., bits: 0s and 1s). It's about turning abstract ideas into structured formats. It's the "What."

Transformation - (How information is transformed)

All computation is transformation of representations by rules. This is the atomic unit of Computation. It's the "How."

Control - (When, Which and Where the Transformation is happening)

Computation is not just what transforms, but the order and conditions under which it happens. It determines the flow of execution.

2. Levels of Abstraction

Organize

  • (R) Bits → Booleans, Integers, Strings
  • (T) Operators (AND, OR, NOT) → Functions
  • (C) Sequences & Cycles → Loops, Recursion, Control Clauses

Optimize

  • (R) Data Structures (Arrays, Linked Lists, Trees, Graphs) → Optimized for transformation
  • (T) Algorithms (Sorting, Searching) → Optimized Transformations
  • (C) Protocols, Programs → Optimized Flow

Scale

  • (R) Structure → Databases, File Systems
  • (T) Computation → Compilers, Interpreters
  • (C) Control → Operating Systems, Networks

3. Domains

Domain Description
Programming Languages Syntax (structure) + Semantics (transformation) + Execution Model (flow)
Operating Systems Manage flow of computation across hardware, structure memory, transform system state
Networking Transmit structured data, transform via protocols, control flow with routing.
Databases Store structured information, transform via queries, control access and transactions
Cryptography Transform information securely (computation), structure keys and messages (info), control access (flow)
